01 — What this is

This is not a coding bootcamp.

You don't need to memorise JavaScript syntax, understand databases, or ever touch a terminal if you don't want to. AI writes the code. You describe what you want.

Think of it like this: you're the architect, the AI is the builder. You say "I need a vocabulary quiz where students type in definitions," and Cursor's AI turns that sentence into a working webpage in about 60 seconds.

What you do need is:

  • A clear idea of what you want to build — even a rough one is fine
  • A GitHub account — free, takes 2 minutes to create
  • Cursor installed — free to start, AI editor that handles the hard parts
  • About 2–3 hours for your first project — less once you've done it once
Good to know Everything in this guide runs in your browser on a laptop or desktop. You don't need a powerful machine — a school-issued Chromebook or an older MacBook works fine.

03 — The stack

Four tools. That's the whole stack.

You don't need ten tools and a subscription manager. Four free (or nearly free) tools cover everything a classroom project needs.

01 / STORAGE + PUBLISHER

GitHub — where your project lives

GitHub stores your code for free and publishes it to the web with one click. Think of it as Google Drive for websites — except it also handles the "put it live" step.

You don't need to understand version control or command-line tools to use GitHub for this. Cursor handles all of that for you. All you need is a free account and a repository (essentially a folder) for each project.

Create a free GitHub account → always free no coding needed
02 / AI EDITOR

Cursor — the AI that writes your code

Cursor is a code editor (like Microsoft Word, but for code) with a built-in AI assistant. You describe what you want in plain English, and the AI writes the code. You review it, then publish.

The free tier is genuinely generous — enough for several classroom projects per month. The AI understands context: if you say "make the quiz timer red when there are 10 seconds left," it knows what that means and changes the right parts automatically.

Heads up Cursor is a desktop app that requires a laptop or desktop computer. It doesn't run on iPads or phones. A school-issued Windows or Mac laptop works great.

03 / FREE HOSTING

GitHub Pages — your live website, no server bill

GitHub Pages takes the files in your GitHub repository and turns them into a real live website at a URL like yourname.github.io/project-name. No hosting bill, no domain purchase required.

This is perfect for static tools — things that don't collect form submissions or store user accounts. A vocab quiz, a class homepage, a grade calculator, a reading list — all of these work perfectly on GitHub Pages for $0.

GitHub Pages docs → completely free HTTPS included

04 — Project ideas

Six projects you can build this weekend.

Each card includes a copy-paste AI prompt you can drop straight into Cursor. The AI will handle the rest — just review and publish.

📝 Vocab Quiz

Students type definitions for a word list you set. Instant feedback, no sign-up required. Works offline once loaded.

Cursor prompt Build a vocabulary quiz webpage. I'll provide a list of words and definitions. Students type the definition, press Enter, and see instant right/wrong feedback with a score counter. No backend — store everything in the HTML file.

📚 Reading Log

A personal reading tracker where students log book title, author, pages read, and a quick rating. Saves in the browser — no account needed.

Cursor prompt Create a reading log webpage. Students can add books with title, author, date finished, pages read, and a 1–5 star rating. Show all entries in a table. Save everything to localStorage so it persists between visits. Include an export to CSV button.

🙋 Volunteer Signup

A simple form that lets parents sign up for classroom events. Submissions go to your email via a free form service.

Cursor prompt Build a volunteer signup form page. Fields: parent name, email, phone (optional), which event they can help with (dropdown), and a notes box. Use Formspree for submissions — I'll add my Formspree endpoint. Style it to look friendly and professional.

🏫 Class Homepage

A dedicated homepage for your class with schedule, upcoming events, resources, and contact info. Looks better than most school portals.

Cursor prompt Create a class homepage for a 5th-grade classroom. Include: a welcome section, weekly schedule table, an upcoming events list, links to resources, and a contact section. Make it colorful and student-friendly. All content in the HTML file so I can edit it directly.

🧮 Grade Calculator

Enter assignment scores and weights, get a final grade instantly. Customisable to match your exact grading rubric.

Cursor prompt Build a weighted grade calculator. Let me define categories (homework, tests, projects) with percentage weights that total 100%. For each category I can enter scores. Show the running weighted average and letter grade in real time as I type.

📓 Homeschool Journal

A private digital portfolio where a student logs daily learning entries with subject, notes, and photos. Saves locally — no cloud account needed.

Cursor prompt Create a homeschool learning journal webpage. Each entry has: date, subject (from a dropdown I can edit), what we covered, how it went (emoji mood picker), and optional notes. Store all entries in localStorage. Show entries in a reverse-chronological feed. Include a simple search bar and a print view.
How to use these prompts Open Cursor, create a new file called index.html, then press Ctrl+K (or Cmd+K on Mac) to open the AI panel. Paste the prompt, press Enter, and watch the code appear. Edit the details (your class name, your word list), save, and push to GitHub Pages.
05 — Your first project

From zero to live webpage, step by step.

This walkthrough builds a vocabulary quiz. The same steps apply to any project — swap in a different prompt at step 5.

  1. Create a free GitHub account Go to github.com and sign up. Use your school email or a personal address — either works. Confirm your email.
    ⏱ 2 min
  2. Create a new repository Click the green "New" button on your GitHub dashboard. Name it something like vocab-quiz. Set it to Public. Check "Add a README file." Click Create repository.
    ⏱ 1 min
  3. Download and install Cursor Go to cursor.com (affiliate link) or cursor.com directly. Download the version for your operating system. Install it — it's just like installing any app.
    ⏱ 5 min
  4. Sign in to GitHub inside Cursor When Cursor opens, click the Source Control icon in the left sidebar (looks like a branching tree). Click "Clone Repository," then "Clone from GitHub." Sign in when prompted — Cursor will remember this.
    ⏱ 2 min
  5. Clone your repository In the Clone Repository dialog, search for vocab-quiz (or whatever you named it). Select it and choose a folder on your computer to save it. Cursor opens the folder automatically.
    ⏱ 1 min
  6. Create a new file and talk to the AI Press Ctrl+N (Windows) or Cmd+N (Mac) to create a new file. Save it as index.html. Now press Ctrl+K or Cmd+K to open the AI chat. Paste the vocab quiz prompt from above — or describe your own idea in plain English.
    ⏱ 2 min
  7. Review and tweak the code The AI will write a complete webpage. You don't need to understand the code — just look at the result in the Preview panel (click the split-screen icon, top right). If something looks wrong, tell the AI: "Make the button bigger" or "Change the colours to blue and white."
    ⏱ 10–20 min
  8. Add your actual content Ask the AI: "Replace the example words with: [your word list here]". Paste in your vocabulary words and definitions. The AI will update the quiz automatically.
    ⏱ 5 min
  9. Push to GitHub In the Source Control sidebar, type a message like "first version" and click the blue Commit button, then Sync. This sends your file to GitHub.
    ⏱ 1 min
  10. Enable GitHub Pages Go to your repository on github.com. Click Settings → Pages (in the left sidebar). Under Source, choose "Deploy from a branch." Select the main branch and the / (root) folder. Click Save. Your site goes live at yourname.github.io/vocab-quiz within about 60 seconds.
    ⏱ 2 min
  11. Share the link Copy the URL from the GitHub Pages settings page and share it with students. Anyone with the link can use the quiz — no app to install, no account to create.
    ⏱ 30 sec
Total time for first project: about 45 minutes. That includes installing Cursor. Future projects take 15–20 minutes once you know the flow.
06 — When to use Railway

Most classroom tools don't need a backend. Here's how to know.

GitHub Pages is perfect for tools that run entirely in the browser. The moment you need to collect or store something server-side, you need a backend. That's where Railway comes in.

GitHub Pages is enough for:

  • Vocab quizzes and flashcards
  • Reading logs (saves in browser)
  • Class homepages and info sites
  • Grade calculators
  • Interactive games and puzzles
  • Homeschool journals (browser storage)

You need Railway when:

  • Collecting form submissions centrally
  • Multiple students share one data source
  • Storing data that must survive browser clears
  • Running a scheduled task (daily email, etc.)
  • Building anything with user accounts
  • Using a database shared across devices

For your first project, start with GitHub Pages only. If you hit a limit — "I want all students to see each other's reading logs" — that's the moment to add Railway. The AI in Cursor can help you add a backend when you're ready; just describe what you need.

08 — FAQ

Honest answers to beginner questions.

Do I need any prior coding knowledge?

No. The AI writes all the code. Your job is to describe what you want in plain English, review the result in a preview window, and say what to change. If it looks right, you're done.

That said, a little curiosity goes a long way. If you find yourself reading the code the AI writes and wondering what things mean, check out the Learn section of this site — it teaches HTML from scratch in plain language.

Is this appropriate for students to use directly?

Cursor is a professional tool aimed at developers, so it's best used by the educator to build tools, not by students. Students then use the finished tools.

If you want students to learn to build things with AI, Hi Bot Code's editor is designed for that. It runs in the browser with no install, and our Learn section walks through building real webpages step by step.

What happens to student data on these tools?

For tools hosted on GitHub Pages (no backend), data is stored in the student's own browser using localStorage. It never leaves their device. There are no accounts, no tracking, and no servers receiving the data.

If you add a backend (Railway + a database), you control the server and the data. You own it — there's no third-party platform seeing your students' information.

Always review your school's data privacy policy before collecting any student data, even in a tool you built yourself.

What if the AI writes code that doesn't work?

This happens sometimes, and it's completely normal. The fix is usually to describe the problem back to the AI: "The timer isn't counting down — can you check the JavaScript?" or "The form submit button isn't doing anything."

Cursor's AI is good at debugging its own code when you describe the symptom. In most cases you'll be back on track in under a minute.

How much does Cursor's free tier include?

The free tier includes a limited number of AI "fast requests" per month (the exact number changes — check cursor.com for current limits). For a few classroom projects per month, the free tier is usually enough.

If you hit the limit, the Pro plan is $20/month. For most educators building and maintaining a small set of tools, the free tier is fine for the first few months.

Can I use a custom domain instead of github.io?

Yes. GitHub Pages supports custom domains. You can purchase a domain for about $10–$15/year from Namecheap, Google Domains, or any registrar, then follow GitHub's instructions to point it at your Pages site.

For most classroom tools, the default yourname.github.io/project URL is perfectly fine. A custom domain is purely cosmetic.

What if I want to update the tool later?

Open Cursor, make your changes with the AI's help (or edit directly if you're comfortable), and push to GitHub again. The live site updates within about 60 seconds. Updates are as easy as the first build.

Is this only for tech-savvy teachers?

No — that's the whole point of this guide. If you can write a clear description of what you want ("a quiz where students match words to pictures"), you have all the skill this requires. The AI handles every technical detail.

The educators who get the most out of this aren't the ones who know the most about computers. They're the ones who know exactly what their students need and can describe it clearly.
